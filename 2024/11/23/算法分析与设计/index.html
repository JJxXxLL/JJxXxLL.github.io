算法分析与设计复习

第一章相关考试内容

  1. 算法和程序的区别：
    • 复习大纲明确提到需要理解算法和程序的差别。
    • PPT中指出：
      • 算法是供人阅读的，解决问题的逻辑过程。
      • 程序是具体实现，用于计算机执行。
  2. 判断问题和优化问题：
    • 复习大纲提到需要理解这两类问题。
    • PPT中列出了三种问题类型：
      • 判断问题：输出是Yes/No，例如某路径长度是否小于k。
      • 优化问题：寻找最优解或最佳值，例如最大独立集问题。
      • 数值计算问题：计算具体数值，例如$\pi$的精确值。
  3. 算法的目标：
    • 设计目标：
      • 易于理解、编写和调试。
      • 高效利用资源。
    • 需要在理解性与高效性之间找到平衡。
  4. 问题分类：
    • PPT中对P、NP、NPC等问题的分类有详细描述，特别强调了以下内容：
      • P：多项式时间内可解的问题。
      • NP：多项式时间内可验证的问题。
      • NPC：所有NP问题都能规约到它的问题。

第二章

五个渐进分析符号的内容：

1. 渐近上界符号 $O(g(n))$

  • 定义：表示函数 $f(n)$ 的增长速度不会超过 $g(n)$ 的一个常数倍，定义为： $O(g(n)) = { f(n) \mid \exists c > 0, \exists n_0 > 0, \forall n \geq n_0, 0 \leq f(n) \leq c \cdot g(n) }$。
  • 直观理解：$f(n)$ 的增长速度在 $g(n)$ 的范围内或更慢。
  • 例子：$f(n) = 3n^2 + 2n + 1 \in O(n^2)$。

2. 渐近下界符号 $\Omega(g(n))$

  • 定义：表示函数 $f(n)$ 的增长速度不会低于 $g(n)$ 的一个常数倍，定义为： $\Omega(g(n)) = { f(n) \mid \exists c > 0, \exists n_0 > 0, \forall n \geq n_0, 0 \leq c \cdot g(n) \leq f(n) }$。
  • 直观理解：$f(n)$ 的增长速度至少和 $g(n)$ 一样快。
  • 例子：$f(n) = 3n^2 + 2n + 1 \in \Omega(n^2)$。

3. 紧渐近界符号 $\Theta(g(n))$

  • 定义：表示函数 $f(n)$ 的增长速度既不会快于也不会慢于 $g(n)$ 的常数倍，定义为： $\Theta(g(n)) = { f(n) \mid \exists c_1, c_2 > 0, \exists n_0 > 0, \forall n \geq n_0, c_1 \cdot g(n) \leq f(n) \leq c_2 \cdot g(n) }$。
  • 直观理解：$f(n)$ 的增长速度和 $g(n)$ 是同一个量级。
  • 例子：$f(n) = 3n^2 + 2n + 1 \in \Theta(n^2)$。

4. 非紧上界符号 $o(g(n))$

  • 定义：表示函数 $f(n)$ 的增长速度比 $g(n)$ 慢得多，$f(n)$ 是 $g(n)$ 的“小无穷小”，定义为： $o(g(n)) = { f(n) \mid \forall c > 0, \exists n_0 > 0, \forall n \geq n_0, 0 \leq f(n) < c \cdot g(n) }$。
  • 等价于：$\lim_{n \to \infty} \frac{f(n)}{g(n)} = 0$。
  • 直观理解：$f(n)$ 的增长速度远远小于 $g(n)$。
  • 例子：$f(n) = n \in o(n^2)$。

5. 非紧下界符号 $\omega(g(n))$

  • 定义：表示函数 $f(n)$ 的增长速度比 $g(n)$ 快得多，$f(n)$ 是 $g(n)$ 的“大无穷大”，定义为： $\omega(g(n)) = { f(n) \mid \forall c > 0, \exists n_0 > 0, \forall n \geq n_0, 0 \leq c \cdot g(n) < f(n) }$。
  • 等价于：$\lim_{n \to \infty} \frac{f(n)}{g(n)} = \infty$。
  • 直观理解：$f(n)$ 的增长速度远远大于 $g(n)$。
  • 例子：$f(n) = n^2 \in \omega(n)$。

符号关系

  • 包含关系： $o(g(n)) \subseteq O(g(n)) \subseteq \Theta(g(n)) \subseteq \Omega(g(n)) \subseteq \omega(g(n))$。
  • 直观图示：
    • $O$ 和 $o$ 表示增长速度的上界（松紧程度不同）。
    • $\Omega$ 和 $\omega$ 表示增长速度的下界（松紧程度不同）。
    • $\Theta$ 表示紧确的增长范围。

渐进分析记号的性质

1. 传递性

  • $f(n) = \Theta(g(n))$ 且 $g(n) = \Theta(h(n)) \Rightarrow f(n) = \Theta(h(n))$
  • $f(n) = O(g(n))$ 且 $g(n) = O(h(n)) \Rightarrow f(n) = O(h(n))$
  • $f(n) = \Omega(g(n))$ 且 $g(n) = \Omega(h(n)) \Rightarrow f(n) = \Omega(h(n))$
  • $f(n) = o(g(n))$ 且 $g(n) = o(h(n)) \Rightarrow f(n) = o(h(n))$
  • $f(n) = \omega(g(n))$ 且 $g(n) = \omega(h(n)) \Rightarrow f(n) = \omega(h(n))$

2. 反身性

  • $f(n) = \Theta(f(n))$
  • $f(n) = O(f(n))$
  • $f(n) = \Omega(f(n))$

3. 对称性

  • $f(n) = \Theta(g(n)) \iff g(n) = \Theta(f(n))$

4. 互对称性

  • $f(n) = O(g(n)) \iff g(n) = \Omega(f(n))$
  • $f(n) = o(g(n)) \iff g(n) = \omega(f(n))$

5. 算术运算

  • 加法：
    • $O(f(n)) + O(g(n)) = O(\max{f(n), g(n)})$
    • $\Theta(f(n)) + \Theta(g(n)) = \Theta(\max{f(n), g(n)})$
  • 乘法：
    • $O(f(n)) \cdot O(g(n)) = O(f(n) \cdot g(n))$
  • 常数因子：
    • $c \cdot O(f(n)) = O(f(n)) \quad (\text{其中 } c > 0)$
  • 若 $g(n) = O(f(n))$，则：
    • $O(f(n)) + O(g(n)) = O(f(n))$

6. 常见公式

  1. 多项式：
    • $a_0 + a_1n + \dots + a_d n^d = \Theta(n^d) \quad (\text{其中 } a_d > 0)$
  2. 对数：
    • $O(\log_a n) = O(\log_b n) \quad (\text{其中 } a, b > 0)$
    • $\log n = O(n^x) \quad (\text{其中 } x > 0)$
  3. 指数：
    • $n^d = O(r^n) \quad (\text{其中 } r > 1, d > 0)$

2.3 三种算法设计技术

1. 贪心算法 (Greedy Algorithm)

  • 基本思想：在每一步中选择“当前最优解”，即局部最优解，期望通过一系列局部最优选择达到全局最优。
  • 局部最优与全局最优的关系：
    • 贪心算法仅在某些特定问题中有效，即局部最优选择能导出全局最优解。
    • 贪心算法通常简单且高效，但并不总是正确。
  • 优点：算法简单、执行效率高。
  • 缺点：可能产生错误结果或次优解。
案例：区间调度问题 (Interval Scheduling)
  • 问题描述：
    • 给定 $n$ 个任务，每个任务有一个开始时间 $s_j$ 和结束时间 $f_j$。
    • 两个任务相容当且仅当它们不重叠。
    • 目标：选择最多数量的相容任务。
  • 贪心策略：
    • 按任务的结束时间 $f_j$ 升序排序。
    • 每次选择当前最早结束且与已选任务不冲突的任务。
  • 时间复杂度：
    • 排序：$O(n \log n)$。
    • 遍历：$O(n)$。
    • 总时间复杂度：$O(n \log n)$。

时间复杂度分析

  1. 排序：将任务按照结束时间排序需要 $O(n \log n)$ 时间。
  2. 遍历：检查所有任务需要 $O(n)$ 时间。

对 区间调度问题 (Interval Scheduling) 的 贪心算法正确性证明 的详细解释。

定理：贪心算法是最优的

证明方法：反证法 (Proof by Contradiction)

  1. 假设贪心算法不是最优的：
    • 设 $i_1, i_2, \dots, i_k$ 是贪心算法选择的任务集合。
    • 设 $j_1, j_2, \dots, j_m$ 是最优解选择的任务集合，且 $m > k$。
    • 假设两个集合的前 $r$ 个任务是相同的，即 $i1 = j_1, i_2 = j_2, \dots, i_r = j_r$，但 $i{r+1} \neq j_{r+1}$。
  2. 分析任务 $i{r+1}$ 和 $j{r+1}$ 的关系：
    • 根据贪心算法的选择策略，任务 $i{r+1}$ 的结束时间比 $j{r+1}$ 的结束时间早，即 $f{i{r+1}} \leq f{j{r+1}}$。
    • 因为贪心算法优先选择结束时间最早的任务，所以任务 $i_{r+1}$ 不与前 $r$ 个任务冲突。
  3. 替换分析：
    • 由于任务 $i{r+1}$ 比 $j{r+1}$ 更早结束，且它与前 $r$ 个任务相容，可以用 $i{r+1}$ 替换 $j{r+1}$。
    • 替换后，新的任务集合仍然是合法的解，且任务数量不变，但结束时间更早。
  4. 矛盾：
    • 根据上述替换操作，可以构造一个比最优解更优的解（即任务数量相同但结束时间更早），这与最优解的定义矛盾。
    • 因此，假设贪心算法不是最优的成立不了。

关键结论

  • 贪心选择性质：每一步选择结束时间最早的任务不会影响全局最优解。
  • 最优子结构：贪心算法的解是最优解的一部分，且每一步选择后剩余问题仍然是原问题的子问题。

2. 分治算法 (Divide and Conquer)

  • 基本思想：将问题分解为多个小规模子问题，递归解决这些子问题，最后合并它们的解以获得原问题的解。
  • 步骤：
    1. 分解 (Divide)：将规模为 $n$ 的问题分解为若干个规模较小的子问题。
    2. 解决 (Conquer)：递归地解决子问题。
    3. 合并 (Combine)：将子问题的解合并为原问题的解。
  • 优点：结构清晰、易于递归实现。
  • 缺点：合并步骤可能复杂，导致效率下降。

分治法乘法 (Divide-and-Conquer Multiplication)

问题描述

如何计算两个 $n$ 位整数的乘积？将问题分解为更小的子问题，通过分治法递归求解。

公式推导

将 $x$ 和 $y$ 分解：设 $x = 2^{n/2} \cdot x_1 + x_0$，其中 $x_1$ 是高位部分，$x_0$ 是低位部分；类似地，设 $y = 2^{n/2} \cdot y_1 + y_0$。

两数乘积公式：
$x \cdot y = (2^{n/2} \cdot x_1 + x_0)(2^{n/2} \cdot y_1 + y_0)$。
展开公式：
$x \cdot y = 2^n \cdot x_1y_1 + 2^{n/2} \cdot (x_1y_0 + x_0y_1) + x_0y_0$。
需要计算以下四个部分：

  1. $x_1y_1$：高位部分的乘积；
  2. $x_0y_0$：低位部分的乘积；
  3. $x_1y_0$：交叉部分的乘积；
  4. $x_0y_1$：另一交叉部分的乘积。

分治法递归

子问题分解：将原问题分解为四个规模为 $n/2$ 的子问题。递归求解：对每个子问题递归计算，直到子问题的规模足够小（例如，$n = 1$）。合并结果：按公式将结果组合起来，最终得到完整的乘积。

时间复杂度分析

递归关系为 $T(n) = 4T(n/2) + \Theta(n)$，其中 $4T(n/2)$ 是四次递归调用的代价，$\Theta(n)$ 是加法和移位操作的代价。使用主方法（Master Method）解递归：$a = 4$，$b = 2$，$\log_b a = \log_2 4 = 2$，时间复杂度为 $T(n) = \Theta(n^2)$

优化：Karatsuba 算法

基本思想是通过减少乘法的次数优化递归步骤。传统方法需要 4 次乘法，而 Karatsuba 方法通过巧妙分组减少为 3 次乘法。Karatsuba 公式为：
$x \cdot y = 2^n \cdot x_1y_1 + 2^{n/2} \cdot (z - x_1y_1 - x_0y_0) + x_0y_0$，其中 $z = (x_1 + x_0)(y_1 + y_0)$。

时间复杂度递归关系为 $T(n) = 3T(n/2) + \Theta(n)$，使用主方法解得 $T(n) = \Theta(n^{\log_2 3}) \approx \Theta(n^{1.585})$。

总结

基本分治法的时间复杂度为 $\Theta(n^2)$，适合初步理解分治法思想；Karatsuba 算法进一步优化，将时间复杂度降为 $\Theta(n^{1.585})$；对于大规模整数乘法，可以使用更高级算法（如傅里叶变换法）进一步优化。

3. 动态规划 (Dynamic Programming)

  • 基本思想：将问题分解为一系列重叠子问题，记录子问题的解以避免重复计算。
  • 步骤：
    1. 分解子问题：将问题分解为若干个相互依赖的子问题。
    2. 状态定义：定义用于表示子问题解的状态。
    3. 状态转移方程：描述子问题之间的关系。
    4. 边界条件：定义初始状态。
案例：背包问题 (Knapsack Problem)
  • 问题描述：
    • 给定 $n$ 个物品，每个物品有重量 $w_i$ 和价值 $v_i$，背包容量为 $W$。
    • 目标：在不超过容量 $W$ 的前提下，使总价值最大。
  • 状态定义：
    • $M[i, w]$ 表示前 $i$ 个物品中，总重量不超过 $w$ 时的最大价值。
  • 状态转移方程：
    • 如果不选第 $i$ 个物品：$M[i, w] = M[i-1, w]$。
    • 如果选第 $i$ 个物品：$M[i, w] = \max{M[i-1, w], v_i + M[i-1, w-w_i]}$。
  • 时间复杂度：$O(nW)$（伪多项式时间）。

总结

  • 贪心算法适合局部最优能够导出全局最优的问题。
  • 分治算法适合子问题独立且可以合并的问题。
  • 动态规划适合存在重叠子问题且最优解可由子问题组合的问题。
